## 三、新买VPS，进后台要做的几件事

买完机器，建议按这个顺序操作：

### 第一步：进 KiwiVM，记下 IP 和 SSH 端口

在主界面（Main Controls）能直接看到。IP 地址和端口，是后续用 SSH 工具连接服务器的必要信息。

### 第二步：重装操作系统

新买的VPS默认装的是 CentOS。考虑到 CentOS 的社区生态已经在走下坡路，官方也停止了维护，**建议第一件事就是换成 Ubuntu 24.04 LTS**。

在 KiwiVM 左侧找到 **Install new OS**，选择 Ubuntu 24.04，点击确认，等几分钟就能完成。重装完成后，页面会显示一串新的 root 密码，**务必复制保存好**，这串密码后续登录服务器要用。

### 第三步：开启二次验证（强烈建议）

KiwiVM 面板的安全性不能马虎。在左侧菜单找到 **Two-Factor Authentication**，用 Google Authenticator 或其他身份验证 App 完成绑定。这样即使账号密码泄露，别人也进不了你的控制面板。

此外，2025 年 KiwiVM 还上线了 **Passkeys（密钥登录）** 功能，支持指纹/面部识别代替密码登录，安全性更高，感兴趣可以在设置里开启。

### 第四步：用 SSH 连接服务器

系统装好之后，就可以通过 SSH 连接到服务器了。

- Windows 用户：用 PuTTY 或 Xshell，填入 IP 地址、SSH 端口，用户名填 `root`，密码就是重装系统时得到的那串
- Mac/Linux 用户：直接在终端输入 `ssh -p [端口号] root@[IP地址]` 即可

连接成功后，终端里出现 `root@xxx:~#` 就说明连上了。

## 四、KiwiVM 后台几个常见操作详解

### 换机房（Migrate to another DC）

这是搬瓦工的招牌功能之一。如果你发现当前机房延迟高、或者IP被污染，可以一键迁移到其他机房，数据全部保留，不会丢失。

操作方法：左侧菜单 → **Migrate to another DC** → 选择目标机房 → 点击 Confirm。通常 1-2 分钟就能完成，迁移后 IP 地址会变，但数据不受影响。

CN2 GIA-E 套餐目前支持在多达 12 个机房之间自由切换，包括美国洛杉矶 DC6/DC9、日本大阪软银、荷兰阿姆斯特丹等，这个灵活度在同价位VPS里很少见。

### 快照备份（Snapshots）

折腾服务器之前，先拍个快照是个好习惯。操作路径：**Snapshots** → Create Snapshot。如果后续操作失败，可以一键恢复到快照时的状态。

### 查看流量使用情况

搬瓦工的流量是按月计算的，超出之后VPS会被暂停到下个月自动恢复，不会产生额外扣费。在主界面可以直接看到本月用了多少、还剩多少，以及流量重置日期。

### AI 智能助理 Assistant Amy

2025 年 KiwiVM 新增了一个挺实用的功能：内置 AI 客服 **Assistant Amy**，支持中文对话。可以用来咨询技术问题、让助理帮你检查 VPS 的连通性问题、或者执行一些自动化操作。对英文不好的用户来说比较友好。

## 六、新手最容易踩的坑

**坑1：忘记保存重装后的 root 密码**

重装系统完成后，KiwiVM 会生成一串新密码，就显示那一次，关闭页面就没了。如果忘了，要再次进入 **Root password modification** 重置。别问我为什么知道。

**坑2：SSH 连接一直超时，以为IP被封**

先冷静。大多数情况不是IP封了，而是防火墙设置问题、SSH服务没起来，或者用的是VPN连不上。先去 KiwiVM 的 **Interactive Shell**（VNC 紧急控制台）看看机器里面的情况，能救回来的概率很大。

**坑3：流量用超了，VPS消失了**

不是消失，是被自动暂停了。流量超出后，VPS会在当月剩余时间内暂停访问，下个月流量重置后自动恢复。在主界面可以看到流量重置日期，心里有数就不会慌。

**坑4：只会从官网后台进KiwiVM，忘了直连地址**

KiwiVM 的直连地址是 `kiwivm.64clouds.com`，建议收藏。但第一次登录要先在 KiwiVM 里设置独立密码，否则直连进去需要输密码但你不知道是什么。
